## Overview

This file implements TaskStore, a class that holds an in-memory Map<string, Task> and exposes create, read, list, update, move and delete operations. Inputs are validated and normalised using sanitise helpers and IDs are created with generateTaskId. Every state-changing operation emits a domain evidence event (task_created, task_updated, task_moved, task_deleted) via emitTaskEvent including actor and from/to metadata.

The store enforces Kanban column validity against KANBAN_COLUMNS, returns shallow copies of Task objects to callers to avoid external mutation, and accepts TaskStoreOptions in the constructor but does not persist state — persistence and subscription to evidence events are intended to be handled by higher-level integrations.

## Architecture Notes

- In-memory authoritative state: tasks are kept in a Map<string, Task> and returned as shallow copies to avoid external mutation.
- Event-driven side effects: every mutation emits a domain-specific evidence event (emitTaskEvent) including actor and from/to metadata to allow external systems to react.
- Input sanitisation & strict column validation: sanitise* helpers are applied to all inputs and KANBAN_COLUMNS is used to ensure only valid KanbanColumn values are accepted; invalid inputs throw errors.

## Usage Examples

### Basic lifecycle (create → update → move → delete)

Instantiate TaskStore via createTaskStore(), call create({ title, column?, ... }) to add a task (generateTaskId is used), use update(id, patch) to change fields (sanitised and validated), call move({ id, toColumn }) to relocate a task between columns (validated against KANBAN_COLUMNS), then delete(id) to remove it. Each mutation emits an evidence event via emitTaskEvent.

## Maintenance Notes

- This store is purely in-memory; constructor options (TaskStoreOptions) are accepted but ignored — persistence must be implemented by higher-level code or adapters.
- Sanitisation functions can throw or return empty values; callers should handle exceptions from create/update/move which throw on invalid input or missing tasks.
- Events include the actor string 'agent:task-manager' hard-coded in the class; if actor attribution must change, update the constructor or emitTaskEvent calls accordingly.

function isValidColumn(col: string): col is KanbanColumn

### Description

Checks whether the provided string exists in the KANBAN_COLUMNS array and acts as a TypeScript type guard narrowing the type to KanbanColumn when true.


The function casts KANBAN_COLUMNS to readonly string[] and uses the Array.prototype.includes method to test if the input string is one of the allowed column identifiers. The implementation is a single expression and does not perform any side effects.
