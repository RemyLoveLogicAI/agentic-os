## Overview

This file provides three small, pure helper functions that coerce unknown values into safe, trimmed string forms before other code operates on them. sanitiseString(value, fallback = "") normalises null/undefined to the fallback and trims inputs; sanitiseOptionalString returns a trimmed string or undefined when input is blank; sanitiseStringArray returns an array of trimmed, non-empty strings, returning [] for non-arrays.

These defensive helpers prevent runtime errors (e.g., calling .trim on undefined), have no dependencies, are synchronous and easy to test, and are intended to be used at the boundary where untyped external input enters the task-manager domain.

## Architecture Notes

- Defensive, boundary-focused design: normalise untrusted/unknown values at the entry point to avoid runtime .trim() errors downstream.
- Pure utility functions with no side effects or external dependencies; easy to test and reuse across request handlers and validation layers.
- Consistent trimming and blank-value handling: blank strings can be mapped to fallback (sanitiseString) or to undefined (sanitiseOptionalString) depending on caller needs.

## Usage Examples

### Main use case

When handling an incoming task creation payload, call sanitiseString for required text fields (e.g., title), sanitiseOptionalString for optional fields (e.g., description) so blank values become undefined, and sanitiseStringArray for tag lists so non-arrays become [] and blank tags are removed. After these calls, downstream code can safely call string methods (like .length or .slice) without guarding for null/undefined.

## Maintenance Notes

- sanitiseString converts non-string primitives using String(value).trim(), which will produce string representations for booleans and numbers (e.g., true -> "true", 123 -> "123"). Consider validating types separately if those should be rejected rather than coerced.
- sanitiseOptionalString returns undefined for empty/whitespace-only inputs. Code that persists data should treat undefined differently from the empty string if that distinction matters.
- sanitiseStringArray ignores non-array inputs and filters out blank entries; callers expecting to preserve non-string array elements should pre-validate the array contents.
