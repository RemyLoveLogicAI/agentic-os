## Overview

This file centralizes the domain type definitions for the task manager. It declares a literal tuple KANBAN_COLUMNS and derives the KanbanColumn union type from it, plus interfaces for Task, TaskCreateInput, TaskUpdateInput, TaskMoveInput, and TaskListFilter to describe task payloads and API-level DTOs. It also defines a KanbanBoard type (Record<KanbanColumn, Task[]>) for board shape and TaskEvent for audit/evidence records.

Persistence-related types are included: PersistMode is a string union ("rip" | "local") and TaskStoreOptions contains optional persist and ledgerPath fields. A file-level comment notes that all string fields are validated at the boundary by sanitiseString so callers won't receive undefined string errors; timestamps and IDs are represented as strings in the interfaces.

## Architecture Notes

- Uses a literal tuple (KANBAN_COLUMNS as const) to derive a KanbanColumn union type, ensuring compile-time enforcement of allowed column names.
- Represents the board structure with TypeScript's Record<KanbanColumn, Task[]> for a fixed-key mapping from column -> list of tasks.
- Separates creation/update DTOs (TaskCreateInput, TaskUpdateInput) from the persisted Task interface to allow optional fields during mutations and full shape for storage/transport.

## Usage Examples

### Create and store a new task

Construct a TaskCreateInput with title (string) and optional description, column (one of KanbanColumn), assignee, and tags; persist code should populate id, createdAt, updatedAt and produce a Task matching the Task interface.

## Maintenance Notes

- All timestamp and identifier fields are declared as strings — callers and persisters should use consistent ISO-formatted timestamps.
- KANBAN_COLUMNS is the single source of truth for allowed column names; add/remove columns there to update the KanbanColumn type.
- PersistMode is a narrow union of literal strings ("rip" | "local"); add values intentionally and update consumers accordingly. ledgerPath is an optional string likely used by local persistence implementations.
