## Overview

Exports a lightweight in-memory event emitter for task state transitions. The module defines a typed listener (EvidenceListener), keeps an internal array of listeners, and exposes onTaskEvent to register a listener (returning an unsubscribe function) and emitTaskEvent to create and broadcast TaskEvent records.

emitTaskEvent constructs a TaskEvent (including an ISO timestamp) and synchronously invokes each registered listener inside a try/catch so listener errors are swallowed. It imports TaskEvent and KanbanColumn types from ./types.js for typing the payload and options. This simple pub/sub is intended for single-process hooks such as appending events to an external evidence ledger.

## Architecture Notes

- Implements a simple in-memory pub/sub pattern: listeners are stored in a module-level array and invoked synchronously on emit.
- emitTaskEvent builds a strongly-typed TaskEvent (including ISO timestamp via new Date().toISOString()) and deliberately swallows listener errors to avoid breaking the caller.
- Because listeners are in-memory and synchronous, this is lightweight but not durable or distributed; it is suitable for single-process usage or as a lightweight hook to append to an external evidence ledger.

## Usage Examples

### Main use case

Register a listener with onTaskEvent(fn) where fn receives a TaskEvent; call emitTaskEvent('moved', 'task-123', 'user@example.com', { from: 'todo', to: 'in-progress', detail: { reason: 'started' } }) to notify all listeners. onTaskEvent returns an unsubscribe function to remove the listener when no longer needed.

## Maintenance Notes

- Listeners are stored in-memory and are not persisted across process restarts; use an external mechanism if persistence or clustering is required.
- emitTaskEvent invokes listeners synchronously and swallows exceptions—listeners should be fast and robust to avoid blocking the emitter.
- Unsubscribing searches by reference (listeners.indexOf(fn)); passing different function instances will not remove previously registered listeners.
