### Description

Settings is a configuration holder class that centralizes application configuration values such as app_name, debug mode, host, port, log_level, and model_config.

Within the module, Settings acts as the centralized configuration class that other parts of the file and package import and use to obtain runtime configuration values. It organizes environment and application parameters in a single, consistent structure.

Responsibilities:
- Define and expose core application configuration values (app_name, debug, host, port, log_level).
- Contain model-specific configuration under the model_config attribute for components that require model parameters.
- Provide a single, typed settings object (via inheritance from BaseSettings) that can be instantiated and validated by the application.
- Serve as the authoritative source of runtime configuration used by services and components in the module.
