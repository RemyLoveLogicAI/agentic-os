## Overview

This module provides an in-memory session store intended as a drop-in swap for a persistent backend like Redis or a database in production. It defines a SessionStore class that manages Session and SessionEvent objects (imported from agentic_ide.models) and exposes a set of methods to create, retrieve, update, and manage sessions and their events. The implementation is designed to keep session state in memory while offering an interface that can be replaced with a persistent store without changing higher-level code.

### Description

SessionStore is responsible for managing persistent session records for the agentic IDE.

Within the services/agentic-ide module, SessionStore acts as the central abstraction for session lifecycle and storage operations, providing a stable API used by other services to persist and query session-related data.

Responsibilities:
- Create new session records and initialize any required session metadata.
- Retrieve session data by identifier for use by other components.
- Append events to an existing session's event history to record actions or changes over time.
- Update the current state of a session as it evolves during use.
- List available session identifiers so callers can discover existing sessions.
