## Overview

This directory contains two service modules used by the agentic IDE: an in-memory session store and a deterministic tool registry. The session_store.py module provides a non-persistent, in-memory session store intended to act as a drop-in alternative during development or testing in place of production backends such as Redis or a database. The tool_registry.py module implements a deterministic mapping from tool names to their corresponding handlers, allowing the system to look up and invoke tools in a predictable manner.

Together these components provide lightweight runtime services: the session store manages ephemeral session state for agents or IDE interactions, while the tool registry centralizes the registration and lookup of tool handlers. This directory is focused on local, development-friendly implementations of services rather than production integrations; the session store is explicitly described as a swap-in for a persistent backend, and the registry provides deterministic behavior for tool dispatch. There are no subdirectories in this folder — the layout is intentionally flat and limited to these two cooperating service modules.


### File Organization

Files are organized flat at the directory root, each implementing a single service responsibility: session_store.py for ephemeral session state and tool_registry.py for deterministic tool-to-handler mapping. This keeps service implementations isolated and easy to replace with production backends.
