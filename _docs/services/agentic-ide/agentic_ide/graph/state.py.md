### Description

WorkflowState is a TypedDict that models the serializable state of a workflow session.

As a TypedDict, WorkflowState provides a structured schema for workflow session data in the module, enabling consistent creation, validation, and transmission of a workflow's runtime state across components. It acts as the canonical payload for persisting or communicating the current state and history of an agentic IDE workflow.

Responsibilities:
- Hold session identity and input: store session_id, command, context, and intent to represent what the workflow is operating on.
- Record safety and decision metadata: carry a risk field (RiskLevel) that represents assessed risk for the current action.
- Describe tool invocation: include tool_name and tool_args to document which tool is being or was used and with what parameters.
- Track execution outcome and progression: maintain status (ActionStatus), result, and error to represent the current execution outcome and any failure information.
- Provide an event history: contain events (list[SessionEvent]) to preserve a chronological record of session-level events for auditing or replay.
