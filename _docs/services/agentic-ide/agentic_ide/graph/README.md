## Overview

This directory holds the core graph-level components used by the Agentic IDE workflow built on top of LangGraph. At the root there are three Python modules: state.py defines the state schema used throughout the Agentic IDE workflow, nodes.py implements standalone node functions that operate on a WorkflowState to perform individual steps, and workflow.py composes those nodes into an assembled LangGraph workflow used by the Agentic IDE command loop. Together these files provide the data model, the processing primitives, and the orchestration that drive the IDE's agentic loop.

The files are organized so that the state schema is centrally defined and referenced by node implementations, and the assembled workflow imports both to construct a runnable LangGraph workflow. This directory's role in the larger system is to encapsulate the graph-based logic for Agentic IDE interactions: state representations (state.py), discrete processing units (nodes.py), and the top-level workflow assembly (workflow.py) which acts as the entry point for the command loop. There are no additional subdirectories under this path.


### File Organization

A flat layout groups three complementary modules at the same level: state.py for the schema, nodes.py for functional node implementations, and workflow.py for assembling the LangGraph workflow. This keeps schema, units of work, and orchestration separated but colocated for easy discovery and maintenance.
