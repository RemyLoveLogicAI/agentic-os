## Overview

This directory holds the HTTP route implementations for the agentic IDE service. At the root level there are two Python modules: health.py, which implements a lightweight health-check endpoint (explicitly for a FastAPI application), and sessions.py, which defines the HTTP route handlers related to session lifecycle and agent command execution for the agentic IDE service. Together these files provide the externally reachable endpoints used to monitor service health and manage interactive agent sessions.

The files are organized to separate a minimal operational endpoint (health.py) from the functional session and command handling (sessions.py). The health-check route is intended to be lightweight and used by orchestration or monitoring systems, while the session routes encapsulate the session lifecycle, command execution, and related HTTP interactions required by the agentic IDE. This directory acts as the HTTP interface layer within the agentic IDE service, presenting endpoints that other services, UIs, or load balancers will call to interact with agents and to check the service status.


### File Organization

Files are organized at the same level with a clear separation of concern: health.py provides a single-purpose health-check endpoint and sessions.py contains the session lifecycle and agent command HTTP handlers. This flat layout keeps small route modules discoverable and focused.
