## Overview

The services directory is the container for service implementations and their tests. At the directory root there are no standalone files; instead service implementations are organized into subdirectories. Each service subdirectory groups the code required to run that particular service including the application entry point, configuration, data models, workflow implementation, HTTP routes, and associated tests as described below.

Currently this directory contains a single subdirectory, agentic-ide, which hosts the Agentic IDE service implementation and its test suite. The agentic-ide subdirectory is organized as a Python package (agentic_ide) and provides modules for application initialization, configuration, data models, workflow code, HTTP routes, and tests. Together these components implement the Agentic IDE service and its verification suite; this services directory's role in the larger system is to collect these service packages so they can be discovered, run, and maintained independently from other parts of the repository.


### File Organization

There are no files at the services root; organization is hierarchical with each service in its own subdirectory. Each service subdirectory contains a package (e.g., agentic_ide) that groups initialization, config, models, workflows, routes, and tests to keep service-specific concerns localized.
