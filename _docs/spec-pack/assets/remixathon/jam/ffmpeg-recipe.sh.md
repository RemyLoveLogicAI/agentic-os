## Overview

A Bash script that automates an ffmpeg pipeline to produce a final submission video. It expects source/master.mp4 (1920x1080@30fps) and optional source/voiceover.wav and source/music.wav. Flags: --preview (faster 480p draft) and --section (render only title or end card); the script exits if master.mp4 is missing.

The script builds title/end card clips from PNGs, programmatically constructs a filter_complex from a hard-coded OVERLAYS array to pre-scale and time PNG overlays (lower-thirds, callouts, a persistent corner bug), renders a body _body.mp4 with mapped audio, and concatenates _title.mp4 + _body.mp4 + _end.mp4 into out/submission-1080p.mp4. Requires ffmpeg >= 5.0 with libx264, libfreetype, and fontconfig.

## Architecture Notes

- Dynamic filter graph: the script builds the ffmpeg filter_complex string programmatically from a hard-coded OVERLAYS array, pre-scaling overlays and chaining overlay labels so each subsequent overlay is applied on top of the previous result.
- Two rendering modes: full (1920x1080, medium preset, CRF 18) and preview (854x480, ultrafast preset, CRF 28) are controlled by --preview which adjusts SCALE_FILTER, OVERLAY_SCALE, PRESET, and CRF.
- Final assembly uses the concat demuxer (text file listing _title.mp4, _body.mp4, _end.mp4) to avoid re-encoding timing mismatches; -movflags +faststart is used to make the MP4 streamable.

## Usage Examples

### Full render (default)

Place source/master.mp4 in source/, optional source/voiceover.wav and source/music.wav if desired. Run: bash ffmpeg-recipe.sh. The script produces out/_title.mp4, out/_body.mp4, out/_end.mp4 and final out/submission-1080p.mp4.

## Maintenance Notes

- Ensure ffmpeg >= 5.0 is installed with libx264, libfreetype, and fontconfig enabled; missing codecs or filters will cause ffmpeg failures.
- Overlay PNG filenames and timings are hard-coded in the OVERLAYS array; if a PNG is missing comment out its entry or add the file. Timing values are relative to master.mp4 start (EDL timestamps in the file subtract 8s as documented).
- The VO and MUSIC variables are defined but not mixed into the final render; to use external audio tracks the filter graph and mapping would need to be extended (current audio map uses -map 0:a?).
