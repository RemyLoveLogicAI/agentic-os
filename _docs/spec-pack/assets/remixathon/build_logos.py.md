## Overview

Defines color tokens, SVG-building primitives, and higher-level composers to produce several logo layouts (primary horizontal, stacked, glyph-only, social card, and a submission badge). The glyph is a programmatic 6-segment circular arc with a central play triangle and supports gradient and monochrome options.

Includes helpers write_svg to persist SVG strings and rasterize to convert SVG to PNG via cairosvg.svg2png. A simple main() build pipeline emits SVG and PNG assets into a LOGO_DIR for CI or local generation.

## Architecture Notes

- SVG generation is split into low-level primitives (loop_glyph, gradient_def, svg_doc) and higher-level composers (primary_horizontal, stacked, glyph_only, social_card, submission_badge) to allow reuse of fragments and variants.
- Build pipeline separates writing SVG (write_svg) and rasterization (rasterize using cairosvg), enabling consumers to generate SVG-only outputs or produce PNGs at different target widths.
- LOGO_DIR is created at module import time (ROOT / 'logo' with mkdir(parents=True, exist_ok=True)), so running the module immediately ensures output directory exists; rasterize will overwrite existing files.

## Usage Examples

### Generate full set of logo assets (command-line)

Run the file as a script (python build_logos.py). main() will create LOGO_DIR, write SVGs for all composers, rasterize PNG versions at configured sizes (via cairosvg), and print a listing of generated files.

### Programmatically obtain an SVG string

Import primary_horizontal/on_dark, stacked, glyph_only, social_card, or submission_badge from this module and call one of them to receive an SVG string. Use write_svg to persist or rasterize to produce PNGs via cairosvg.svg2png.

## Maintenance Notes

- Requires the third-party cairosvg package for rasterization; if cairosvg is missing, rasterize() will fail. Consider making rasterization optional or reporting a clear error for CI environments.
- The text elements rely on system fonts (DejaVu Sans, Inter, Helvetica, Arial); rendering may differ across platforms if these fonts are not available. For deterministic output consider embedding fonts in SVG or ensuring font availability in build environments.
- Be aware of the unused import 'os' (can be removed) and that LOGO_DIR is created on import; if you prefer lazy creation, move LOGO_DIR.mkdir() into main or write_svg.
