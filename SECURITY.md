# Security Policy

Last updated: Thursday, April 30, 2026 (PDT)

This repository is committed to responsible disclosure and rapid mitigation of security vulnerabilities.

---

## Reporting a security issue

Preferred method:
- Use GitHub Security Advisories to submit a private vulnerability report (if enabled for this repository).

Alternative method:
- If Security Advisories are not available, open a private vulnerability report via GitHub’s security reporting flow (if supported by your account), or open an issue that describes the impact without providing exploit steps.

---

## What to include

Please include:
- Clear description of the vulnerability
- Minimal reproduction path (if safe)
- Impact assessment (what an attacker can do)
- Affected versions/commit SHAs

Avoid:
- Public disclosure of exploit steps before the issue is triaged and mitigations are prepared
- Uploading sensitive credentials or secrets

---

## Response timeline

We will:
- Acknowledge receipt as soon as possible
- Assess severity and determine fix/mitigation
- Coordinate with the reporter if follow-up details are needed

---

## Disclosure policy

We aim to publish security fixes promptly after assessment. Public disclosure generally occurs after:
- mitigations are available, or
- a security advisory is required to protect users.
