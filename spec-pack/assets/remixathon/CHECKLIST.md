### How to use

```bash
# 1) Drop your master screen recording
cp ~/Desktop/jam-master.mp4 jam/source/master.mp4

# 2) Render
cd jam && bash ffmpeg-recipe.sh --preview   # quick draft
bash ffmpeg-recipe.sh                        # final 1080p

# 3) Submit
ls out/submission-1080p.mp4
```

The ffmpeg recipe was validated end-to-end against a synthetic stand-in
master.mp4 — all 16 overlays (15 timed + 1 persistent corner bug) composite
cleanly with the right timecodes. See `jam/EDL.md` for the master timeline.

## ⏳ Remaining (handed back to operator)

These three items require operator input that Devin cannot create on its own.
Everything graphical is done — the operator just needs to drop in source media:

- [ ] **Master screen recording** — operator captures one ~2:42 take using the
      command in `jam/README.md`. Drop at `jam/source/master.mp4`.
- [ ] **Voiceover narration** — record per the "Voiceover beat" column in
      `jam/EDL.md`. Drop at `jam/source/voiceover.wav` (optional; `master.mp4`
      audio is used by default).
- [ ] **Music underscore** — soft synth pad. Drop at `jam/source/music.wav`
      (optional, mixed at –18 dB by the recipe).

Once those three are in place, `bash jam/ffmpeg-recipe.sh` produces the final
`jam/out/submission-1080p.mp4` that's ready to upload to the Remixathon.

## Build / regeneration

```bash
cd spec-pack/assets/remixathon
python3 build_logos.py          # regenerates logo/
python3 build_demo_graphics.py  # regenerates demo/
python3 build_jam_assets.py     # regenerates jam/ overlays + cards
```

Requirements: Python 3 with `cairosvg` (`pip install cairosvg`) and a system
font that resolves "DejaVu Sans" (already present on Devin VMs and most Linux
distros).
