# Contributing to Agentic OS

Thank you for your interest in contributing.

## How to Contribute

### 1. Read the Spec

Start with:
- `README.md` — overview and quickstart
- `spec-pack/master-spec.md` — canonical contract
- `spec-pack/phase-0-build-plan.md` — current build focus

### 2. Open an Issue

Before submitting a pull request, open an issue to discuss:
- the problem you're solving
- the proposed approach
- any trade-offs or alternatives

### 3. Submit a Pull Request

When ready:
- fork the repo
- create a feature branch
- make focused, atomic commits
- reference the issue in your PR description
- ensure all markdown renders correctly

## Style Guidelines

### Markdown
- Use clear headings
- Use tables for structured data
- Use code fences for examples
- Keep paragraphs concise

### Code
- Follow existing patterns
- Comment non-obvious logic
- Prefer small, composable functions
- Add tests where applicable

### Diagrams
- Use D2 syntax for architecture diagrams
- Store source `.d2` files alongside rendered `.png`
- Keep diagrams minimal and readable

## Governance

All contributions are reviewed against the system's core principles:
- No proof, no claim.
- Prefer reversible decisions.
- Keep the first slice thin and real.
- Treat memory as a product surface.
- Treat governance as a feature.

## Code of Conduct

Be respectful, constructive, and collaborative. See `CODE_OF_CONDUCT.md` for details.

## Questions

Open a Discussion or Issue, or reach out to the maintainers directly.

Thank you for helping build the Agentic Web.
